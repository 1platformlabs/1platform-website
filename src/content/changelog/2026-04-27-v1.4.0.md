## API Developer Documentation

Eight new integration flows landed alongside the latest API release, each documenting an end-to-end developer workflow with full request/response examples sourced from the OpenAPI spec.

- **Dashboard Overview** — Bootstrap a fully branded dashboard from a single API call: branding, theme palette, layout, modules, feature flags, supported languages, and home KPIs (balance, unread notifications, resource counters). Includes the i18n bundle endpoint with `If-None-Match` 304-aware caching.
- **Dashboard Settings** — Manage app-level key/value settings (announcement banners, feature toggles, runtime configuration) via list, upsert, read-by-key, and delete endpoints. `PUT` is idempotent for safe retries.
- **Magic Link Authentication** — Passwordless email sign-in: request a magic link, exchange the single-use token for an access + refresh token pair, rotate refresh tokens on demand, and revoke sessions on logout.
- **Notifications** — In-app notification feed with severity (`info`, `success`, `warning`), per-user or global broadcast targeting, mark-as-read, and a dedicated unread-count endpoint for navbar badges.
- **Referrals** — Resolve a referral code to its destination payload (`code`, `valid`, `referrer_username`, `signup_url`) so public landing pages can render personalized copy and attribute signups.
- **Support** — Embed a support center: list and create tickets with priority and status, append threaded user/agent replies, and serve a public FAQ catalog (with admin-only publishing).
- **Tasks** — Personal task list with `todo` / `in_progress` / `done` lifecycle, optional due dates, and automatic `completed_at` tracking when status moves to `done`.
- **Webhook Configuration** — Configure outbound webhooks: register allowed domains (returns the initial signing secret), declare per-event handler URLs (`on_approved`, `on_denied`, `on_cancelled`, `on_expired`, `on_dismiss`), remove domains, and rotate the HMAC signing secret.

## API

- Added the `GET /api/v1/plans` endpoint to list active billing plans (`free`, `payg`, `subscription`) with currency, price, and billing cycle metadata. The Payments & Subscriptions flow now includes a dedicated step to enumerate plans before initiating an upgrade.
