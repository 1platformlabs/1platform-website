## Fixed

- **Authentication field name** — All docs code examples now use the correct `apiKey` field (previously `api_key`) matching the OpenAPI spec.
- **Token response field** — Updated all examples to reference `data.access_token` (previously `data.token`) with full response schema including `token_type` and `expires_in`.
- **Dual-token auth** — Added the required `x-user-token` header to all code examples (Python, JavaScript, cURL) that previously only showed app token authentication.
- **Keyword extraction field** — Fixed JavaScript example to use `lang` instead of `language`, matching the API schema.
- **Content generation request** — Removed non-existent `website_id` field from content generation examples, replaced with correct `lang` and `country` parameters.

## Pages Updated

- `docs/getting-started` — Added user token step, fixed auth field names
- `docs/authentication` — Added full response schemas, fixed field names, added expiration details
- `docs/code-examples` — Complete rewrite aligning all three languages (Python, JS, cURL) with the current OpenAPI spec
